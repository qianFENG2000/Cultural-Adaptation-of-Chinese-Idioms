README for eval/

This folder stores the automatic evaluation outputs and the derived summary files used for analysis, tables, and figures.

The folder is organized into three main parts:

eval/
├── task1/
├── task2/
└── merged/


1. Purpose of this folder
-------------------------

The eval/ folder contains the results of automatic evaluation for the round-trip translation experiments.

The automatic evaluation is conducted after the model outputs have been generated and cleaned. The evaluation scripts compare source texts, intermediate English translations, and final Chinese back-translations under different evaluation settings.

The main evaluation directions are:

- zh_en: Chinese input compared with English translation
- en_zh: English translation compared with Chinese back-translation
- zh_zh: original Chinese input compared with final Chinese back-translation


2. eval/task1/
--------------

This folder stores automatic evaluation results for Task 1.

Task 1 is the idiom-level round-trip translation task:

Chinese idiom -> English translation -> Chinese idiom

The files in this folder are generated by:

python scripts/run_evaluation.py

Typical files include:

- task1_detail_{eval_type}_{model}_{zh_en_prompt_version}_{en_zh_prompt_version}.json
- task1_summary_{eval_type}_{zh_en_prompt_version}_{en_zh_prompt_version}.json

The detail files contain item-level evaluation results. They usually include:

- index
- reference
- prediction
- included_in_eval
- skip_reason
- BLEU
- ROUGE
- BERTScore
- SBERT cosine similarity
- COMETKiwi
- MetricX-QE
- COMET

The summary files contain model-level average scores for each evaluation setting. They are later merged into a single CSV file by merge_summary_auto_eval.py.


3. eval/task2/
--------------

This folder stores automatic evaluation results for Task 2.

Task 2 is the sentence-level round-trip translation task:

Chinese sentence containing an idiom -> English translation -> Chinese sentence

Task 2 contains three back-translation conditions:

- C1 / v3_v1: natural Chinese sentence
- C2 / v3_v2: Chinese sentence with an idiom
- C3 / v3_v3: Chinese sentence with the original idiom

The files in this folder are generated by:

python scripts/run_evaluation.py

Typical files include:

- task2_detail_{eval_type}_{model}_{zh_en_prompt_version}_{en_zh_prompt_version}.json
- task2_summary_{eval_type}_{zh_en_prompt_version}_{en_zh_prompt_version}.json

As in Task 1, the detail files contain item-level metric scores and skip information, while the summary files contain model-level average scores.


4. eval/merged/
---------------

This folder stores merged and processed automatic evaluation results.

It is generated after the Task 1 and Task 2 evaluation files have been created.

Important files may include:

- summary_auto_eval.csv
- auto_eval_full_table.csv
- grouped_table_index.csv
- task-specific grouped CSV tables
- invalid and wrong-language summary CSV files
- auto_pic/ figures

The files in eval/merged/ are mainly generated by the following scripts:

python scripts/merge_summary_auto_eval.py
python scripts/export_auto_eval_tables.py
python scripts/plot_auto_eval_figures.py
python scripts/summarize_invalid_and_wrong_language.py


5. Main files in eval/merged/
-----------------------------

summary_auto_eval.csv

This file is generated by:

python scripts/merge_summary_auto_eval.py

It merges all automatic evaluation summary JSON files from eval/task1/ and eval/task2/ into one long-format CSV file.

It is the main intermediate summary file for automatic evaluation.


auto_eval_full_table.csv

This file is generated by:

python scripts/export_auto_eval_tables.py

It is a cleaned table version of summary_auto_eval.csv and keeps the most important columns for reporting, such as:

- task
- eval_type
- zh_en_prompt_version
- en_zh_prompt_version
- model
- num_valid
- BLEU
- BERTScore
- SBERT cosine similarity
- COMETKiwi
- MetricX-QE
- COMET


grouped_table_index.csv

This file is generated by:

python scripts/export_auto_eval_tables.py

It records the grouped automatic evaluation tables created for each task, evaluation type, and prompt setting.


Task-specific grouped CSV files

These files are also generated by:

python scripts/export_auto_eval_tables.py

They split the automatic evaluation results by task, evaluation type, and prompt setting.

Example naming pattern:

{task}_{eval_type}_{zh_en_prompt_version}_{en_zh_prompt_version}.csv


Invalid and wrong-language summary files

These files are generated by:

python scripts/summarize_invalid_and_wrong_language.py

They summarize invalid outputs and wrong-language outputs across tasks, models, and evaluation settings.

They are useful for analyzing model compliance and output quality beyond automatic metric scores.


6. eval/merged/auto_pic/
------------------------

This folder stores automatic evaluation figures.

It is generated by:

python scripts/plot_auto_eval_figures.py

Typical output figures include:

- fig1_auto_zh_en.pdf
- fig2_auto_en_zh.pdf
- fig3_auto_zh_zh.pdf

These figures visualize automatic evaluation scores across models and task conditions.


7. Recommended generation order
-------------------------------

The recommended order for generating files in eval/ is:

Step 1. Run automatic evaluation

python scripts/run_evaluation.py

This creates detail and summary JSON files in:

- eval/task1/
- eval/task2/


Step 2. Merge automatic evaluation summaries

python scripts/merge_summary_auto_eval.py

This creates:

- eval/merged/summary_auto_eval.csv


Step 3. Export automatic evaluation tables

python scripts/export_auto_eval_tables.py

This creates:

- eval/merged/auto_eval_full_table.csv
- eval/merged/grouped_table_index.csv
- grouped task-specific CSV files


Step 4. Plot automatic evaluation figures

python scripts/plot_auto_eval_figures.py

This creates figures under:

- eval/merged/auto_pic/


Step 5. Summarize invalid and wrong-language outputs

python scripts/summarize_invalid_and_wrong_language.py

This creates CSV files summarizing invalid outputs and wrong-language outputs.


8. Notes
--------

The evaluation scripts expect cleaned model outputs as input.

The cleaned output files should usually be stored under:

- outputs/task1/
- outputs/task2/

and should end with:

_clean.jsonl

For example:

- outputs/task1/task1_en_zh_llama31_v1_v1_clean.jsonl
- outputs/task2/task2_en_zh_llama31_v3_v1_clean.jsonl
- outputs/task2/task2_en_zh_llama31_v3_v2_clean.jsonl
- outputs/task2/task2_en_zh_llama31_v3_v3_clean.jsonl

If these cleaned files do not exist, the automatic evaluation scripts cannot run successfully.
