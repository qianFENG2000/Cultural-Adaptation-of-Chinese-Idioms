README for outputs/

This folder stores model-generated translation outputs for the main experiments.

The folder is organized as follows:

outputs/
├── task1/
└── task2/


1. Purpose of this folder
-------------------------

The outputs/ folder contains the raw and cleaned outputs generated by the large language models.

These outputs are produced before automatic evaluation and human evaluation.

The general workflow is:

1. Run model translation with scripts/run_translation.py
2. Clean the generated outputs manually
3. Use cleaned outputs for automatic evaluation
4. Use cleaned outputs to build human evaluation datasets


2. outputs/task1/
-----------------

This folder stores outputs for Task 1.

Task 1 is the idiom-level round-trip translation task:

Chinese idiom -> English translation -> Chinese idiom

The first step translates a standalone Chinese idiom into English.

The second step translates the English output back into Chinese, with the goal of producing a Chinese idiom.

Typical files include:

- task1_zh_en_{model}_{zh_en_prompt_version}.jsonl
- task1_zh_en_{model}_{zh_en_prompt_version}_clean.jsonl
- task1_en_zh_{model}_{zh_en_prompt_version}_{en_zh_prompt_version}.jsonl
- task1_en_zh_{model}_{zh_en_prompt_version}_{en_zh_prompt_version}_clean.jsonl

Example filenames:

- task1_zh_en_llama31_v1.jsonl
- task1_zh_en_llama31_v1_clean.jsonl
- task1_en_zh_llama31_v1_v1.jsonl
- task1_en_zh_llama31_v1_v1_clean.jsonl

The most important cleaned file for later evaluation is usually:

task1_en_zh_{model}_v1_v1_clean.jsonl


3. outputs/task2/
-----------------

This folder stores outputs for Task 2.

Task 2 is the sentence-level round-trip translation task:

Chinese sentence containing an idiom -> English translation -> Chinese sentence

The first step translates a Chinese sentence containing an idiom into English.

The second step translates the English output back into Chinese under different prompting conditions.

The main Task 2 conditions are:

- C1 / v3_v1: natural Chinese sentence
- C2 / v3_v2: Chinese sentence with an idiom
- C3 / v3_v3: Chinese sentence with the original idiom

Typical files include:

- task2_zh_en_{model}_{zh_en_prompt_version}.jsonl
- task2_zh_en_{model}_{zh_en_prompt_version}_clean.jsonl
- task2_en_zh_{model}_{zh_en_prompt_version}_{en_zh_prompt_version}.jsonl
- task2_en_zh_{model}_{zh_en_prompt_version}_{en_zh_prompt_version}_clean.jsonl

Example filenames:

- task2_zh_en_llama31_v3.jsonl
- task2_zh_en_llama31_v3_clean.jsonl
- task2_en_zh_llama31_v3_v1.jsonl
- task2_en_zh_llama31_v3_v1_clean.jsonl
- task2_en_zh_llama31_v3_v2.jsonl
- task2_en_zh_llama31_v3_v2_clean.jsonl
- task2_en_zh_llama31_v3_v3.jsonl
- task2_en_zh_llama31_v3_v3_clean.jsonl

The most important cleaned files for later evaluation are usually:

- task2_en_zh_{model}_v3_v1_clean.jsonl
- task2_en_zh_{model}_v3_v2_clean.jsonl
- task2_en_zh_{model}_v3_v3_clean.jsonl


4. File naming convention
-------------------------

The output filenames follow this general pattern:

{task}_{direction}_{model}_{prompt_version}.jsonl

or, for round-trip files:

{task}_{direction}_{model}_{zh_en_prompt_version}_{en_zh_prompt_version}.jsonl

Where:

- task: task1 or task2
- direction: zh_en or en_zh
- model: model identifier
- zh_en_prompt_version: prompt version used for Chinese-to-English translation
- en_zh_prompt_version: prompt version used for English-to-Chinese back-translation

Common model identifiers are:

- llama31
- mistral
- deepseek
- gemma2
- glm4
- qwen25


5. Raw files and cleaned files
------------------------------

Raw output files are generated directly by:

python scripts/run_translation.py

Raw files may contain invalid outputs, wrong-language outputs, prompt leakage, repeated text, explanations, or other formatting issues.

Cleaned files end with:

_clean.jsonl

Cleaned files are required for later evaluation scripts.

The cleaned files should contain additional fields such as:

- is_invalid_en
- invalid_type_en
- language_status_en
- is_invalid_zh
- invalid_type_zh
- language_status_zh

The cleaning step is necessary because automatic and human evaluation scripts rely on these fields to decide which outputs are valid and how invalid outputs should be handled.


6. Important fields
-------------------

The exact fields may differ depending on task and direction, but the main fields usually include:

- index
- source_idiom
- sentence
- translation_en
- translation_zh
- is_invalid_en
- invalid_type_en
- language_status_en
- is_invalid_zh
- invalid_type_zh
- language_status_zh

For Task 1:

- source_idiom is the original Chinese idiom.
- translation_en is the English translation.
- translation_zh is the Chinese back-translation.

For Task 2:

- sentence is the original Chinese sentence containing the idiom.
- source_idiom is the idiom contained in the sentence.
- translation_en is the English translation.
- translation_zh is the Chinese back-translation.


7. Recommended generation order
-------------------------------

The recommended order for generating files in outputs/ is:

Step 1. Generate Task 1 Chinese-to-English outputs

python scripts/run_translation.py --task task1 --direction zh_en --model all --prompt_version v1

This creates files under:

outputs/task1/


Step 2. Clean Task 1 Chinese-to-English outputs

Clean the generated files and save them as:

task1_zh_en_{model}_v1_clean.jsonl


Step 3. Generate Task 1 English-to-Chinese outputs

python scripts/run_translation.py --task task1 --direction en_zh --model all --zh_en_prompt_version v1 --prompt_version v1

This creates files under:

outputs/task1/


Step 4. Clean Task 1 English-to-Chinese outputs

Clean the generated files and save them as:

task1_en_zh_{model}_v1_v1_clean.jsonl


Step 5. Generate Task 2 Chinese-to-English outputs

python scripts/run_translation.py --task task2 --direction zh_en --model all --prompt_version v3

This creates files under:

outputs/task2/


Step 6. Clean Task 2 Chinese-to-English outputs

Clean the generated files and save them as:

task2_zh_en_{model}_v3_clean.jsonl


Step 7. Generate Task 2 English-to-Chinese outputs for C1

python scripts/run_translation.py --task task2 --direction en_zh --model all --zh_en_prompt_version v3 --prompt_version v1

This creates:

task2_en_zh_{model}_v3_v1.jsonl


Step 8. Generate Task 2 English-to-Chinese outputs for C2

python scripts/run_translation.py --task task2 --direction en_zh --model all --zh_en_prompt_version v3 --prompt_version v2

This creates:

task2_en_zh_{model}_v3_v2.jsonl


Step 9. Generate Task 2 English-to-Chinese outputs for C3

python scripts/run_translation.py --task task2 --direction en_zh --model all --zh_en_prompt_version v3 --prompt_version v3

This creates:

task2_en_zh_{model}_v3_v3.jsonl


Step 10. Clean all Task 2 English-to-Chinese outputs

Clean the generated files and save them as:

- task2_en_zh_{model}_v3_v1_clean.jsonl
- task2_en_zh_{model}_v3_v2_clean.jsonl
- task2_en_zh_{model}_v3_v3_clean.jsonl


8. Downstream usage
-------------------

The cleaned files in outputs/ are used by automatic evaluation scripts:

python scripts/run_evaluation.py

They are also used by the human evaluation preparation script:

python scripts/build_idiom_human_eval_dataset.py

Therefore, do not delete cleaned files after generation.


9. Notes
--------

The outputs/ folder should be treated as a generated output folder.

Raw files can be regenerated by running scripts/run_translation.py.

Cleaned files are more important for analysis because they are used in both automatic evaluation and human evaluation.

If a downstream script fails because a file is missing, first check whether the required *_clean.jsonl file exists in outputs/task1/ or outputs/task2/.
