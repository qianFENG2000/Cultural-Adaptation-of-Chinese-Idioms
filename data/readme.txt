README for data/
================

This folder contains the input datasets used in the Chinese idiom round-trip translation project.

The files are organized in the following order:

1. idiom_base.json
2. idiom_sample.json
3. idiom_sample_ann.json
4. idiom_eval_human.json


1. idiom_base.json
------------------

Description:
    This is the main idiom dataset used for the translation experiments.

Number of items:
    905

Format:
    JSON array

Fields:
    sentence:
        A Chinese sentence containing the idiom.

    index:
        The original numeric ID of the idiom item.

    source_idiom:
        The source Chinese idiom.

Example item:
    {
      "sentence": "她喜欢打扮，愿意有朋友，可是这都不过是一些小小的、哀而不伤的、青春的游戏",
      "index": 1,
      "source_idiom": "哀而不伤"
    }

Used by:
    scripts/run_translation.py

Main role:
    This file is the full dataset for Task 1 and Task 2 translation generation.

In the current project:
    Task 1 uses source_idiom as the input for idiom-level translation.
    Task 2 uses sentence as the input for sentence-level translation.


2. idiom_sample.json
--------------------

Description:
    This is a random sample of 100 idioms selected from idiom_base.json.

Number of items:
    100

Format:
    JSON array

Fields:
    sentence:
        A Chinese sentence containing the idiom.

    index:
        The original numeric ID inherited from idiom_base.json.

    source_idiom:
        The source Chinese idiom.

Generated by:
    scripts/run_sample.py

Input source:
    data/idiom_base.json

Used for:
    Manual annotation before human evaluation sampling.

Main role:
    This file is an intermediate sample file. It is not the final human evaluation sample.
    It should be manually annotated with polarity and transparency labels to create idiom_sample_ann.json.


3. idiom_sample_ann.json
------------------------

Description:
    This is the annotated version of idiom_sample.json.

Number of items:
    100

Format:
    JSON array

Fields:
    sentence:
        A Chinese sentence containing the idiom.

    index:
        The original numeric ID inherited from idiom_base.json.

    source_idiom:
        The source Chinese idiom.

    polarity:
        The manually annotated sentiment or evaluative polarity of the idiom.
        Possible values in this file:
            positive
            neutral
            negative

    transparency:
        The manually annotated semantic transparency of the idiom.
        Possible values in this file:
            high
            low

Distribution:
    Polarity:
        positive: 39
        neutral: 23
        negative: 38

    Transparency:
        high: 75
        low: 25

    Polarity + transparency groups:
        negative + high: 30
        negative + low: 8
        neutral + high: 16
        neutral + low: 7
        positive + high: 29
        positive + low: 10

Created from:
    data/idiom_sample.json

Used by:
    scripts/run_sample_eval.py
    scripts/summarize_human_eval.py

Main role:
    This file provides annotation labels for the sampled idioms.
    It is used to select a balanced subset for human evaluation and to add metadata back to human evaluation results.


4. idiom_eval_human.json
------------------------

Description:
    This is the final idiom subset used for human evaluation.

Number of items:
    12

Format:
    JSON array

Fields:
    sentence:
        A Chinese sentence containing the idiom.

    index:
        The original numeric ID inherited from idiom_base.json.

    source_idiom:
        The source Chinese idiom.

    polarity:
        The manually annotated sentiment or evaluative polarity of the idiom.

    transparency:
        The manually annotated semantic transparency of the idiom.

Generated by:
    scripts/run_sample_eval.py

Input source:
    data/idiom_sample_ann.json

Used by:
    scripts/build_idiom_human_eval_dataset.py

Main role:
    This file defines the idioms included in the human evaluation set.
    The script build_idiom_human_eval_dataset.py uses these idioms to extract corresponding model outputs and generate human evaluation files.

Distribution:
    Polarity:
        positive: 4
        neutral: 4
        negative: 4

    Transparency:
        high: 6
        low: 6

    Polarity + transparency groups:
        negative + high: 2
        negative + low: 2
        neutral + high: 2
        neutral + low: 2
        positive + high: 2
        positive + low: 2


Recommended data pipeline
-------------------------

The recommended data preparation order is:

    Step 1:
        Start from idiom_base.json.

    Step 2:
        Run:

            python scripts/run_sample.py

        This creates:

            data/idiom_sample.json

    Step 3:
        Manually annotate idiom_sample.json with polarity and transparency.

        Save the annotated file as:

            data/idiom_sample_ann.json

    Step 4:
        Run:

            python scripts/run_sample_eval.py

        This creates:

            data/idiom_eval_human.json

    Step 5:
        Use idiom_base.json for full automatic translation experiments.

    Step 6:
        Use idiom_eval_human.json for human evaluation dataset construction.


Relationship between files
--------------------------

The relationship between the files is:

    idiom_base.json
        |
        | generated by scripts/run_sample.py
        v
    idiom_sample.json
        |
        | manually annotated with polarity and transparency
        v
    idiom_sample_ann.json
        |
        | sampled by scripts/run_sample_eval.py
        v
    idiom_eval_human.json


Important notes
---------------

1. Do not overwrite idiom_base.json unless the full experimental dataset changes.

2. idiom_sample.json is an intermediate file. It does not contain polarity or transparency labels.

3. idiom_sample_ann.json should be kept carefully because it contains manually added annotation labels.

4. idiom_eval_human.json is the final human evaluation idiom subset.

5. The current idiom_eval_human.json contains 12 idioms because scripts/run_sample_eval.py uses SAMPLE_SIZE = 2 for each polarity-transparency group.

6. If a larger human evaluation set is needed, change SAMPLE_SIZE in scripts/run_sample_eval.py before generating idiom_eval_human.json.

7. The index field should be preserved across all files. It is used to match items between datasets and model output files.

8. The source_idiom field is the main idiom-level input for Task 1.

9. The sentence field is the main sentence-level input for Task 2.
